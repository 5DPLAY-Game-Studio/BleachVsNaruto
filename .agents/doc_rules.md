# `.agents` 文档写法（AI）

读者：仅 AI。`AGENTS.md` 与 `.agents/*.md` 均非给人读。在 `.agents/` **新建或改写**时遵守本文；人读说明见 `CHANGELOG.md` / `HOW2BUILD.md` / `README.md`。

现有：`AGENTS.md`（入口索引）← `modules.md` `map.md` `code_style.md` `comment.md` `codegen.md` `bat_script.md` `jsfl.md` `doc_rules.md`。

---

## MUST

| # | 规则 |
|---|------|
| 1 | 标题带用途标签，如 `（AI）` / `速查（AI）`，开篇 1～2 句说明**何时读、解决什么** |
| 2 | 面向决策与执行：硬规则、对照表、最短示例；禁止教程腔、故事腔、重复堆砌 |
| 3 | 能表则表；`MUST` / `NEVER` / 主题→入口 优先于长段落 |
| 4 | 约定完整：精简不删语义；拿不准就保留原约束并链到源（如 `Project.xml`、ASDoc） |
| 5 | 单文件只服务一个目的；文件名短、小写、下划线：`topic.md` |
| 6 | 放在 `.agents/` **根下**（除非用户明确要求子目录） |
| 7 | 交叉引用用相对路径；被 `AGENTS.md` 需要发现的，在 `AGENTS.md` 加一行链接 |
| 8 | 行分隔符 `CRLF`；中文正文；代码块标明 `actionscript` 等 |
| 9 | 示例代码须符合 [`code_style.md`](code_style.md)；注释示例符合 [`comment.md`](comment.md) |
| 10 | 索引类文档（如 [`map.md`](map.md)）只记入口/边界，不写类级流水账缓存 |

## NEVER

- 为人写的完整教程、大段「为什么」、空话模板（设计理念 N 条等）
- 与 `AGENTS.md` 或其它 `.agents` 文件大段复制同一内容（改一处链过去）
- 无必要的每类一文件「记忆缓存」
- 过期仍留错入口（发现漂移立即改索引，勿另开影子文档）
- 把根目录人读文档（README / CHANGELOG）的冗长结构强加到本目录 AI 文档

## 推荐骨架

```markdown
# 主题（AI）

一句话：何时读 + 约束来源（可选链接）。

---

## MUST
| # | 规则 |
|---|------|
| 1 | ... |

## NEVER
- ...

## 模式速查 / 对照表
（最短代码或表）

## （可选）源配置键
（与外部文件逐项对应时才写）
```

速查/地图类可更短：开篇 + 1～2 张表即可（参见 [`map.md`](map.md)）。

## 选型

| 需求 | 做法 |
|------|------|
| 格式/注释硬约束 | 独立规范文件（已有 style/comment） |
| 生成结构/命名/单例/协作方式 | [`codegen.md`](codegen.md) |
| 模糊问题定位 | 单文件短表索引（已有 map）；默认只覆盖主战场模块 |
| 隐性坑、禁止事项 | 可短文或并入 map/相关规范；少而准 |
| 类职责说明 | 写进源码 ASDoc，不进 `.agents` 缓存 |

新建前先问：能否扩现有文件？能则不新建。
