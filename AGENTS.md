# AGENTS.md

## 项目信息

这是一个名为 **死神vs火影** 的 2D 格斗游戏。使用 ActionScript 3 语言编写，Adobe AIR 作为运行时。

项目整体采用 **代码 + 美术** 分离开发：
代码技术栈主要使用 Intellij IDEA + Flash/Flex 插件进行开发，同时也兼容 VSCode（或基于其的其他智能编辑器） + ActionScript & MXML 插件的形式开发。
美术技术栈使用 Adobe Animate 进行导出 `BleachVsNaruto_FlashSrc` 目录下的 Flash 源文件。

## 项目核心架构及简要说明

- `BleachVsNaruto_FlashSrc` 游戏所有美术资源存放目录，通常情况下，这里仅涉及美术开发，除非特殊要求，在进行代码开发的时候，请尽量避免在此目录下进行长时间任务。
- `CORE_Components` 游戏的可交互组件，其作用是作为 `BleachVsNaruto_FlashSrc/SRC_XFL/#swc/component/component.xfl` 文件的代码类资源使用，它会编译成可供 Adobe Animate 中可使用的 **组件库**。它的使用频率较低，基本是面向使用 Adobe Animate 进行人物或 UI 编辑时才会使用。目前开发重心并不在此，常规游戏性内容开发也不涉及此处。
- `CORE_KernelLogic` 游戏的核心逻辑，此模块是项目的重点，存放了所有关于游戏核心玩法（例如角色战斗，游戏玩法）的逻辑，建议详细阅读此模块，充分理解该模块的所有内容，生成足够使用的上下文，游戏开发 90% 都会聚焦在此模块。
- `CORE_Shared` 游戏的共享组件，对外接口，存放游戏需要对外暴露的部分常量和共用方法，同时也存放游戏的版本记录，不同版本的 `CORE_Shared` 组件可以通过检查版本来确保版本一致。该模块同时也是 `BleachVsNaruto_FlashSrc/SRC_XFL/#swc/shared/shared.xfl` 文件的代码资源类，在编译 BleachVsNaruto_FlashSrc 的资源时会自动导出到`shared/lib/swc/ide/shared.swc` 文件。
- `CORE_Utils` 游戏的部分公用工具，将部分常用方法提取，供项目的其他模块调用。

## 代码风格

请确保生成的代码与项目整体的代码风格保持一致。以下是从 `.idea/codeStyles/Project.xml` 解析出的详细代码风格规范：

### 引号与分号
- **单引号优先**：字符串统一使用单引号 `'`，不使用双引号 `"`
- **强制分号**：所有语句必须以分号 `;` 结尾

### 大括号与换行
- **强制大括号**：所有控制流语句（if/while/for/do-while）必须使用大括号，即使只有一行语句
- **控制关键字换行**：else、catch、finally 关键字单独起一行
- **括号换行**：方法调用、方法定义、数组初始化等的左右括号各占一行

### 对齐与缩进
- **对齐对象属性**：对象字面量属性对齐
- **对齐变量声明**：变量声明对齐
- **对齐多行链式调用**：链式方法调用对齐
- **对齐多行参数**：方法调用和定义的多行参数对齐
- **对齐多行操作符**：二进制、三元操作符换行后对齐
- **case 不缩进**：switch 语句中的 case 不额外缩进

### 换行与折行
- **对象字面量换行**：对象字面量换行时左右大括号各占一行
- **赋值换行**：赋值语句较长时等号后换行
- **注释换行**：长注释自动换行
- **数组初始化换行**：数组初始化换行时左右大括号各占一行

### 其他规范
- **C风格注释格式化**：自动格式化块注释
- **行分隔符**：使用 CRLF（Windows 风格）作为行分隔符

通常项目负责人会使用 IntelliJ IDEA 对代码进行格式化，请遵循以上规范。

## 生成代码要求

请确保生成的代码除必须的字符串与注释资源外，尽量使用符合 ActionScript 3 的官方代码规范，如果在 ActionScript 3 上没有找到类似规范，可以延伸使用 `JavaScript` 或 `Java` 语言的部分规范，生成的代码建议对必要的逻辑部分添加详尽的注释声明。
如果生成的是整段整段的完整方法，请根据 ActionScript 3 的 ASDoc 规范对该方法添加详细的 ASDoc 注释，同理可参考 Java 语言的 JavaDoc 规范。
